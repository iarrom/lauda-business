# Lauda Business

Бизнес-репозиторий стартапа **Lauda** — AI SaaS для инфлюенс-маркетинга на международном рынке.

Здесь живёт всё, что не код: стратегия, исследования рынка, сегменты, конкуренты, продажи, продукт, найм, финансы.

Репозиторий организован как **самообучающаяся LLM Wiki** по методу Andrej Karpathy (см. [`CLAUDE.md`](./CLAUDE.md)). Сырые источники — в `raw/`, структурированная синтезированная wiki — в `wiki/`. LLM-агент (Claude / Codex / Cursor) поддерживает wiki в актуальном состоянии.

---

## Быстрый старт для нового члена команды

1. Открой папку `lauda-business/` в Obsidian (как vault) или VS Code.
2. Начни с [`wiki/overview.md`](./wiki/overview.md) — высокоуровневая картина бизнеса.
3. Если ищешь что-то конкретное — открой [`wiki/index.md`](./wiki/index.md), это каталог.
4. Хронология последних событий — в [`wiki/log.md`](./wiki/log.md).
5. Прежде чем работать с LLM-агентом, прочти [`CLAUDE.md`](./CLAUDE.md) — это правила игры.

## Как добавить новое знание

**Не редактируй wiki/ руками.** Wiki поддерживает LLM, чтобы избежать рассинхронизации.

1. Положи сырой источник (статья, транскрипт интервью, скриншот, отчёт) в соответствующую подпапку `raw/`.
2. Скажи агенту: `Ingest raw/<путь к файлу>`.
3. Агент создаст summary в `wiki/sources/`, обновит связанные страницы концептов/сегментов/конкурентов, обновит `index.md` и `log.md`.
4. Сделай `git commit` с понятным сообщением.

## Структура

```
raw/                — immutable исходники. LLM только читает.
  interviews/       — транскрипты JTBD / switch / экспертных интервью
  meetings/         — записи команды, звонков с клиентами
  competitors/      — материалы конкурентов (сайты, скриншоты, фичи)
  market-research/  — внешние отчёты, статьи о рынке инфлюенс-маркетинга
  customer-feedback/— фидбэк от клиентов, NPS, тикеты
  strategy-docs/    — внешние стратегические документы
  product-feedback/ — пользовательские отзывы о Lauda
  financial/        — сырые финансовые данные
  assets/           — картинки, скриншоты

wiki/               — LLM-generated, синтезированное знание
  index.md          — каталог всех страниц
  log.md            — хронология ingest/query/lint
  overview.md       — высокоуровневая картина бизнеса
  concepts/         — общие концепты (PMF, JTBD, метрики, юнит-экономика)
  segments/         — клиентские сегменты по AJTBD (Замесин)
  competitors/      — страницы конкурентов
  customers/        — карточки реальных клиентов и интервьюируемых
  product/          — продуктовые решения, фичи, гипотезы
  sales/            — pipeline, активные сделки
  marketing/        — каналы, кампании, контент
  team/             — команда, найм, роли
  finance/          — метрики, юнит-экономика, выручка
  decisions/        — ADR (Architecture/Strategic Decision Records)
  sources/          — summary каждого сырого источника
  comparisons/      — сравнения (мы vs конкуренты, варианты решений)

prompts/            — переиспользуемые промпт-шаблоны для команды
scripts/            — вспомогательные скрипты (импорт, экспорт, бэкап)
```

## Конвенции коммитов

```
ingest: <название источника>      — добавил новый источник в raw/, wiki обновлена
query:  <тема>                    — зафайлил результат запроса в wiki
lint:   <дата>                    — здоровье wiki, исправления
update: <страница>                — точечное обновление wiki руками
chore:  <что>                     — служебные (структура, .gitignore, README)
```
